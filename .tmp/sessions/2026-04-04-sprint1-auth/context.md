## Architecture Decision: Dual Auth

- **Teachers/Admin**: Better Auth session cookie (`better-auth.session_token`)
  - Middleware: `betterAuthMiddleware` reads cookie or Authorization Bearer, attaches `user` to Hono context
  - Always followed by `requireRole('teacher')` or `requireRole('admin')`
- **Students**: Bearer JWT in `Authorization` header (custom, NOT Better Auth)
  - Middleware: `studentAuthMiddleware` reads `Authorization: Bearer <token>`, verifies with `env.JWT_SECRET`
  - Attaches `student` payload to Hono context
- **Never mix middleware** — student routes ONLY use studentAuthMiddleware; teacher/admin routes ONLY use betterAuthMiddleware + requireRole

## Project Conventions (from AGENTS.md + living-notes.md)

- **Runtime**: Bun (use `Bun.password.hash/verify` for PIN, NOT bcrypt directly)
- **Imports**: Always `@/` path alias — NEVER relative `../../` paths
- **Error classes**: Always use typed errors from `@/shared/lib/errors.ts` — NEVER `throw new Error(...)`
- **Responses**: Always use `successResponse` / `errorResponse` from `@/shared/lib/response.ts` — NEVER hand-craft JSON
- **Validation**: Always use `@hono/zod-validator` with `zValidator('json', schema)` — NEVER trust raw `c.req.json()`
- **Naming**: Files kebab-case, functions camelCase, Types PascalCase, schemas camelCase+Schema suffix, constants SCREAMING_SNAKE_CASE
- **Indentation**: 2 spaces, double quotes, semicolons always, trailing commas in multiline
- **No `any`**: Use `unknown` and narrow explicitly
- **Schema barrel**: Every new `*.table.ts` MUST be re-exported from `src/db/schema.ts`
- **Better Auth FK type**: `varchar(36)` — foreign keys referencing Better Auth user IDs must match
- **studentId**: School-assigned string like `"2024-0001"` — NOT a UUID
- **Soft deletes**: Prefer `deleted_at timestamp` over hard delete for audit trail
- **Tests**: Colocated `{feature}.test.ts`, use `app.request()` — no real server needed, mock external services

## Rate Limiting Implementation

For student login rate limiting (5 requests/min per studentId):
- Use Redis INCR + EXPIRE pattern
- Key: `rate_limit:{identifier}:{windowStart}`
- Window: 60 seconds
- Limit: configurable per middleware call

```ts
// Usage in route
app.post('/login', rateLimit({ limit: 5, windowMs: 60_000, key: (c) => c.req.valid('json').studentId }), ...)
```
